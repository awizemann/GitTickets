GitTickets — view redesign

Reference render · neutral default theme (system blue) · maps 1:1 to the SwiftUI handoff
How to read this. Both screens use a clean, brand-neutral native style — no design-system identity — and are fully driven by GitTicketsTheme (accent, fonts, corner radius, header icon). The demo accent is the platform-default system blue, the most compatible choice for a drop-in package; in a host app it inherits that app's accent. The form reads top-to-bottom as a single story — privacy → what's wrong → what we'll send → submit — and the privacy banner + consent toggle share one visual language so they feel like one trust flow.
